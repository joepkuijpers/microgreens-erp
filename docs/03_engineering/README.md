# Engineering

## Purpose

The Engineering domain describes every physical and technical system that supports the Microgreens ERP project.

Unlike the Biological Production domain, which focuses on understanding living systems, Engineering focuses on designing, building, operating and maintaining the technologies that observe and support those systems.

Engineering exists to serve biology.

Technology is never the objective.

It is a tool that helps us understand biological processes, improve cultivation, reduce waste and build a more transparent food chain.

---

## Philosophy

The engineering philosophy of this project can be summarized in one sentence:

> Technology should adapt to biology, not biology to technology.

Every engineering decision should therefore answer one or more of the following questions:

- Does it improve our understanding of biological processes?
- Does it reduce unnecessary labour?
- Does it increase transparency?
- Does it improve sustainability?
- Does it help future generations build upon this knowledge?

Technology that cannot justify its contribution should not be added simply because it is available.

---

This domain includes all technical systems used throughout the project, including:

Each document explains the design decisions, implementation strategy and long-term vision for that subject.

---

## Design Principles

Engineering throughout this project follows several core principles.

### Biology First

Engineering supports biological understanding rather than replacing it.

### Modular Design

Every component should be replaceable without redesigning the entire system.

### Scalability

Solutions should work for one rack today and many racks in the future.

### Simplicity

The simplest reliable solution is preferred over unnecessary complexity.

### Standardisation

Where possible, the same connectors, communication protocols and installation methods should be used throughout the project.

### Maintainability

Systems should be designed so they can be repaired, calibrated and upgraded throughout their lifetime.

---

## Relationship with Other Domains

Engineering does not operate independently.

It continuously exchanges information with the other knowledge domains.

### Project Foundation

Provides the philosophy and long-term direction that guide engineering decisions.

### Biological Production

Defines what biological processes require monitoring or control.

### Information Systems

Stores, analyses and visualises engineering data through the ERP.

### Operations

Describes installation, maintenance, calibration and operational procedures.

## Long-Term Vision

The Engineering domain is intended to evolve alongside the project.

Initially it documents a single Raspberry Pi controlling one cultivation rack.

Over time it should grow into a complete engineering reference for larger biological production systems, integrating monitoring, automation, research equipment and future technologies into one coherent architecture.

Every engineering decision should contribute to a system that becomes more understandable, more maintainable and more sustainable with every generation.
