# Data Strategy

## Purpose

The Data Strategy defines how data should be collected, structured, preserved, and transformed into knowledge throughout the Microgreens ERP project.

Data exists to improve understanding of the food chain, support evidence-based decision-making, and enable continuous improvement.

The objective is not to collect as much data as possible.

The objective is to collect meaningful data with sufficient context to generate reliable knowledge.

## Vision

Every observation contributes to understanding.

Every measurement contributes to knowledge.

Every decision should be supported by reliable information whenever possible.

The long-term ambition is to build a food chain in which data continuously improves biological understanding, operational performance, sustainability, and scientific research.

## Data Philosophy

Data should never exist in isolation.

Every record should describe a real-world event, object, process, or relationship within the food chain.

Data should not only describe what happened.

It should preserve enough context to understand why it happened and improve what happens next.

## Data Development

## Data Architecture

The operational database is based on a relational SQL database.

A relational database has been selected because it represents real-world relationships in a structured, reliable, and scalable manner.

Tables represent entities.

Relationships represent interactions between those entities.

As technology evolves, the SQL implementation may change, but the principles of relational modelling, data integrity, and traceability should remain unchanged.

## Data Principles

Data should be:

- Accurate
- Complete whenever possible
- Traceable
- Time stamped
- Consistent
- Verifiable
- Reproducible
- Understandable
- Maintainable

## Traceability

Every important event within the food chain should remain traceable.

Whenever practical, records should preserve their relationships rather than becoming isolated data points.

Historical information should remain available for future analysis.

## Historical Preservation

Historical data should not be deleted simply because it is no longer operationally active.

Historical records provide the foundation for:

- Trend analysis
- Scientific research
- Continuous improvement
- Traceability
- Auditability

## Data Quality

Incorrect data is often more harmful than missing data.

Whenever uncertainty exists, it should be clearly identified rather than hidden.

Validation should improve reliability without preventing future learning.

## Experimental and Production Data

The project distinguishes between production data and experimental data.

Experimental observations support research.

Production data supports daily operations.

Both contribute to the long-term knowledge of the project.

## Sensor Integration

Sensor observations should be stored with sufficient context to allow future interpretation.

Whenever possible, measurements should be linked to:

- Time
- Batch
- Crop
- Location
- Sensor
- Environmental conditions

## Relationship with the ERP

The ERP preserves operational knowledge by connecting observations, production records, traceability, and research into one integrated system.

Its purpose is not simply to store information.

Its purpose is to help transform data into understanding.

## Long-Term Vision

The long-term ambition is to create a data platform that continuously improves the understanding of the food chain.

Technology, biology, and human knowledge should strengthen one another through reliable information, transparent relationships, and evidence-based decision-making.
